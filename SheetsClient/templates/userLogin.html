This is a Prototype that I have developed using the MEAN stack, in particular Angular2 for the Front End.
The Client, a Financial Institution, wants to launch on the market a new Family of products dedicated exclusively to the Digital channel. Each of these products represent a bundle of financial instruments (stocks, bonds, funds) selected by the Financial Institution around a certain investment theme (e.g. Automotive in East Asia or Energy in North America). Inspiration for this family of Products has been derived from Motif (www.motifinvesting.com).
In a series of workshops where the client has presented the main characteristics of the products and how the presale/sale processes should be developed on the Digital channel.
Starting from the what has emerged from the Workshops, leveraging Motif as an example, I have developed the Prototype.

The prototype has been developed with Angular2 and Typescript for the front end, Express/NodeJS to build a simple light server using MongoDB as DB.
We need still to work on ‘Responsiveness’ (i.e. make the site work optimally on different devices which have different screen sizes) and multi browser support (now the prototype works best with Chrome and last Safari).
A part from these 2 points, we think that the Prototype in its front end (Angular2) part is 'production quality' code. We leverage greatly the composition pattern (via Angular2 components) and have based the solution on a sound Object Oriented model (made possible by the use of Typescript).

You can use any credential to login.
If you fill both the PFA and Customer IDs then it means that you are working as a PFA who wants to prepare a proposal for a Customer.
If you fill just the Customer ID (leaving the PFA ID empty) then it means that you are entering as a customer and you are entitle to view the proposals that the PFA has prepared for you.

The prototype covers the following functionalities:
- Dashboard (basically the Product Catalogue): in this page all the available products are shown with different search and sorting criteria
- Detailed View: shows the details of a single product (e.g. description, financial data, charts, assets that compose the product); through this view we can also customize the product (within the limits set by the min/max rules) and save the customized version.
- Compare view: we can compare 2 Products
- Proposal view: starting from the Details view we can create a Proposal (if we are acting as PFA)
- Proposal list and detail view: if we enter as customer, we can see the list of our proposals, we can select one, modify the amount of investment and, eventually, decide to buy. The process ends with the creation of a message which can go to the bace end system responsible for the execution of the orders on the market.
